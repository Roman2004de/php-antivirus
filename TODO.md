Критичные рекомендации
Убрать утечку debug-информации из AJAX
В admin/ajax.php при ошибке наружу отдаются exception, file, line. Для антивирусного модуля это плохо: раскрывает пути и структуру проекта. Оставить только код ошибки, подробности писать в лог.
Доработать CLI-сканер
install/tools/scan.php сейчас фактически skeleton: возвращает skeleton_ready, но не запускает реальное сканирование. Нужно сделать полноценный CLI-интерфейс:
--path, --profile, --action, --dry-run, --json, --report, --batch-size.
Внешние сигнатуры есть, но не подключены
SignatureLoader::loadFromFile() реализован, но в Scanner используется только loadDefaultRules(). Нужно добавить опцию signatures_path и реально мержить внешние правила с дефолтными.
Режим delete заявлен, но не реализован
В настройках есть delete, в ScanConfig есть ACTION_DELETE, но applyConfiguredAction() его не выполняет и возвращает unsupported. Лучше либо убрать из UI, либо реализовать безопасное удаление только через quarantine-like metadata + dry-run protection.
Нужна защита от параллельных сканов
Сейчас можно стартовать несколько сканов одновременно. Добавить lock-файл/статус активной сессии, иначе будут гонки, нагрузка на диск и конфликт с карантином.
Важные улучшения
Сделать нормальный audit-log
Логировать:
кто запустил скан,
путь,
профиль,
действие,
найденные угрозы,
quarantine/restore/delete,
ошибки.
Лучше отдельный AuditLogger, не только JSON-отчеты.
Усилить карантин
Сейчас файл переносится в payload.bin, это хорошо. Но стоит добавить:
checksum до/после перемещения,
права 0600 на директории/метаданные,
запрет restore в критичные системные файлы без подтверждения,
журнал restore/delete.
Проверить права модуля
Сканирование требует W, просмотр результатов R. Это логично. Но операции quarantine restore/delete должны требовать максимум прав, желательно W или отдельное право уровня администратора.
Улучшить исключения путей
Сейчас FileFilter::isExcluded() использует strpos($path, $excludePath) !== false. Это может давать ложные совпадения. Лучше сравнивать как path-prefix с нормализацией:
путь равен исключению или начинается с excludePath . '/'.
Добавить лимиты на размер session/report
Сессия хранит список всех файлов и все результаты в одном JSON. На больших проектах Bitrix это может стать огромным файлом. Лучше хранить:
список файлов отдельно,
результаты append-only,
summary отдельно.
Добавить прогресс без предварительного хранения всего списка файлов
Сейчас startScan() сначала собирает все файлы в массив. На больших сайтах это долго и дорого по памяти. Лучше сделать iterator-based сканирование или предварительную индексацию чанками.
Добавить scan profiles для Bitrix
Например:
quick: /upload, /bitrix/php_interface, /local/php_interface, /local/modules;
standard: весь document root с исключениями;
deep: включая .txt, .sql, .svg, .htaccess, подозрительные расширения.
Улучшения детектора
Добавить контекстные правила
Сейчас scoring суммирует сигнатуры. Нужны compound-правила:
base64_decode + eval,
gzinflate + create_function,
$_REQUEST + assert,
file_put_contents + .php,
chmod + upload.
Добавить AST/token-анализ для PHP
Regex хорошо как первый слой, но для PHP лучше добавить token_get_all():
искать динамические вызовы,
переменные функции,
конкатенацию payload,
include из переменных,
obfuscated strings.
Добавить entropy-анализ
Для webshell/backdoor полезно считать entropy длинных строк и suspicious encoded payload.
Добавить baseline/изменения файлов
Для Bitrix очень полезно:
hash baseline,
новые/измененные PHP-файлы,
изменения в /bitrix/php_interface/init.php,
/local/php_interface/init.php,
/upload.
Добавить whitelist не только по finding, но и по trusted vendor paths
Например исключать штатные директории модулей, но не полностью, а с отдельным профилем доверия.
Готовность к маркетплейсу
Убрать inline CSS из admin/scan.php
CSS уже есть в /install/css/admin.css. Лучше перенести стили туда.
Добавить полноценные lang-файлы для всех страниц
Проверить, чтобы все сообщения были через Loc::getMessage(), без hardcoded English/Russian в PHP.
Добавить README для администратора
Нужно описать:
что модуль ищет,
что такое dry-run,
как работает карантин,
где хранятся отчеты,
какие директории исключены,
как восстановить файл из карантина.
Добавить миграцию/обновления
Для маркетплейса нужны аккуратные версии и update-скрипты, особенно если появятся новые опции, таблицы или структура отчетов.
Добавить автотесты без Bitrix-зависимости
Хорошо, что есть smoke-тесты. Я бы вынес ядро сканера в максимально независимый слой и добавил PHPUnit:
rules,
whitelist,
quarantine,
report writer,
file filter,
external signatures.
Техническое резюме

Синтаксических ошибок PHP я не нашел. Архитектурно модуль уже выглядит как рабочий MVP, но для production/маркетплейса я бы в первую очередь исправил: утечку debug-информации, внешний файл сигнатур, настоящий CLI-режим, блокировку параллельных сканов, хранение больших отчетов и честную реализацию/удаление режима delete.
